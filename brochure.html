Pourquoi Course-as-Code

Un cours Academy n'est plus une suite de clics dans une UI. C'est un fichier course.yaml versionné dans Git, appliqué à LearnWorlds via une commande unique. Édition, revue, rollback, fork, environnement multiple : tous les outils de l'ingénierie logicielle, appliqués à la production pédagogique.

Les 7 commandes du CLI

Commande
Rôle
course apply
Applique un course.yaml. Idempotent. Flags : --update, --prune, --reorder.
course pull
Reverse : exporte un cours live en course.yaml pour édition locale.
course clone
Deep-clone avec substitution texte : --replace 'old=>new'.
course diff
Diff structuré live ↔ live, ou live ↔ YAML, exit code 1 si divergence.
auth refresh-cookies --launch
Login Playwright headed → snapshot storage-state.
courses get | users | bundles | ...
94 endpoints CRUD générés depuis le spec OpenAPI.
course apply --include-units
Active la couche unités (API privée + cookies + CSRF + form-urlencoded).

Workflow typique

1. Pull un cours existant

learnworlds-pp-cli course pull --slug module-0-onboarding-introduction-15-min --out module-0.yaml

2. Édite le YAML dans Claude Code / VSCode

Renommer une section, ajouter une unité type: embed qui pointe vers un Gist HTML5 animé, modifier le quiz, supprimer un module obsolète.

3. Diff avant push (sécurité)

learnworlds-pp-cli course diff --a-file module-0.yaml --b-slug module-0-onboarding-introduction-15-min

4. Apply

learnworlds-pp-cli course apply --from module-0.yaml --confirm --include-units --update --reorder

Dynamic content

Toute production HTML générée par un skill Claude Code (skill demo-interactives, hyperframes, d3-viz, gsap) atterrit dans LearnWorlds via une unit type: embed. Le champ embed_html accepte un iframe pointant vers un Gist, GitHub Pages, Observable, ou directement du HTML inline via srcdoc.
